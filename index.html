Welcome to my notes

I frequently find myself looking up the same terminal commands when deploying new Ubuntu/Debian servers. To save time, I compiled this quick reference guide.

This page acts as a central hub for basic operations ranging from network configuration to process management. It is written purely in HTML/CSS with no external dependencies.

Why Ubuntu?

Most of these commands apply to any Debian-based distribution. Ubuntu Server simply provides a vast repository and excellent community support, making it my default choice for headless environments.

File Management

Basic commands for navigating the file system and managing permissions.

Viewing & Finding

List all files, including hidden ones, with human-readable sizes:

ls -lah

Find a file by name starting from the root directory:

find / -name "nginx.conf" 2>/dev/null

Permissions

Give the owner read/write/execute permissions, and others read/execute:

chmod 755 /path/to/script.sh

Change the owner of a directory recursively:

chown -R www-data:www-data /var/www/html

Networking & UFW

Securing the server using the Uncomplicated Firewall (UFW) and checking ports.

Basic UFW Setup

A standard configuration that blocks everything except SSH, HTTP, and HTTPS:

ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable

Checking Connections

Check which services are listening on which ports:

ss -tulpn

Or using the older netstat tool:

netstat -plnt

System & Processes

Monitoring resource usage and managing system services.

Disk and Memory

Check free disk space on mounted drives:

df -h

Check available RAM and Swap:

free -m

Managing Services (Systemd)

Restart Nginx and check its status:

systemctl restart nginx
systemctl status nginx

Enable a service to start automatically on boot:

systemctl enable fail2ban
